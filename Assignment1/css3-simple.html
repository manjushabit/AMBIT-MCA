§1Brief History of CSS

CSS (Cascading Style Sheets) was proposed by Håkon Wium Lie in 1994 while he worked at CERN. Before CSS, web pages were styled using HTML attributes — mixing content with presentation, which was messy and hard to maintain.

1996 — CSS Level 1
The First Standard

Published by W3C. Covered basic properties: fonts, colors, text alignment, margins, padding, borders. Supported by IE 3 and Netscape. Very limited — no positioning or layout tools.

1998 — CSS Level 2
Positioning & Media Types

Added absolute/relative/fixed positioning, z-index, media types (print, screen), pseudo-elements (::before, ::after), and the display property. Browser support was inconsistent — the "CSS wars" began.

2004–2011 — CSS 2.1
Stability & Bug Fixes

CSS2 was revised to fix inconsistencies. The box model hack era — different browsers calculated width differently, requiring workarounds. IE6 had its own broken box model.

2011–Present — CSS Level 3
Modular & Powerful

CSS3 is NOT one big document — it is split into independent modules, each developed separately: Selectors, Animations, Transforms, Flexbox, Grid, Custom Properties, etc. This allows browsers to implement features one at a time.

2012 — Flexbox
Layout Revolution

Flexible Box Layout solved the age-old problem of vertical centering and equal-height columns — things that previously required hacks with tables, floats, and negative margins.

2017 — CSS Grid
True Two-Dimensional Layout

CSS Grid gave developers a proper two-dimensional layout system, replacing float-based grid frameworks (Bootstrap, Foundation) with native browser capabilities.

2017 — CSS Custom Properties
Variables in CSS

CSS variables (--color: #333) arrived natively, removing the need for preprocessors like SASS/LESS for basic variable management. Live updates via JavaScript too.

2023–Now — CSS Level 4+
Cascade Layers, :has(), Container Queries

New features: @layer for cascade control, the :has() parent selector (the most-requested feature ever), and container queries (respond to element size, not viewport size).

Key insight: CSS went from basic text formatting (CSS1) → layout tools (CSS2) → rich visual effects and proper layout systems (CSS3) → component-level responsive design (CSS4). Each version solved problems the previous one could not.

§2CSS3 Basics

A CSS rule has two parts: a selector (what to target) and a declaration block (what to apply). Each declaration is a property–value pair.

/* Anatomy of a CSS rule */
h1 {                        /* selector */
  color:       #2563eb;   /* property: value */
  font-size:   2rem;
  font-weight: 700;
}

/* Three ways to add CSS */
<link rel="stylesheet" href="style.css">   /* external ✓ best */
<style> body { background: #fff; } </style>    /* internal */
<p style="color:red">...</p>                  /* inline (avoid) */

/* CSS Custom Properties (Variables) */
:root {
  --brand-color: #2563eb;
  --gap:         1rem;
}
button { background: var(--brand-color); }
Specificity — Who Wins?

When two rules target the same element, the one with higher specificity wins.

* 0
p 1
.class 10
#id 100
style="" 1000
!important overrides everything. Use only as a last resort.
Cascade & Inheritance

Cascade means styles flow from multiple sources and the most specific wins.

SourcePriority
Browser defaultLowest
User stylesheetLow
Author CSS (your file)Normal
Author !importantHigh
User !importantHighest

Inheritance: some properties (font, color, line-height) are automatically passed from parent to child. Others (margin, padding, border) are not inherited.

§4The Box Model

Every HTML element is a rectangular box. The box has four layers from inside out:

MARGIN
BORDER
PADDING
CONTENT
Margin
outside space
Border
visible edge
Padding
inner space
Content
text / image
/* All four layers */
.box {
  width:   200px;    /* content width */
  height:  100px;

  padding: 20px;     /* inside space (all 4 sides) */
  border:  2px solid #333;
  margin:  30px;     /* outside space */
}

/* box-sizing: the most important CSS rule to add globally */
* { box-sizing: border-box; }
/* border-box: width includes padding + border  ← recommended */
/* content-box: width is content only (default) ← confusing   */

/* Shorthand examples */
margin:  10px 20px;         /* top/bottom  left/right */
padding: 10px 20px 15px 5px; /* top right bottom left */

§5Positioning

The position property controls how an element is placed on the page. The default is static — elements sit in the normal document flow.

ValueBehaviourtop/left/right/bottom?
staticNormal flow — default for all elementsNo effect
relativeOffset from its own normal position. Space still reserved.Yes — offset from normal
absoluteRemoved from flow. Positioned relative to nearest positioned ancestor.Yes — relative to parent
fixedRelative to the viewport. Stays on scroll.Yes — relative to screen
stickyLike relative until scroll threshold, then sticks like fixed.Yes — threshold value

Container (position: relative)

Box A
Box B (static)
Box B sits in normal flow after Box A. The top / left properties have no effect here.

Container (position: relative)

Box A
Box B (relative)
Box B is shifted top: 14px; left: 20px from where it would normally sit. The original space is still reserved.

Container (position: relative)

Box A
Box B (absolute)
Box B is taken out of the flow and placed at top: 8px; right: 12px of the nearest position: relative parent (the stage).
Fixed positioning is relative to the browser viewport — the element stays in place when you scroll. Used for sticky navigation bars and floating action buttons. Cannot be shown inline here without covering the page.
.navbar {
  position: fixed;
  top:       0;
  left:      0;
  width:     100%;
  z-index:   1000;   /* stays on top */
}
Sticky acts like relative until the element reaches a scroll threshold (top: 0), then it behaves like fixed. Great for table headers and sidebar nav.
.section-header {
  position: sticky;
  top:       0;       /* sticks when it reaches top of viewport */
  background: white;  /* needs bg or content shows through */
  z-index:   10;
}

§6Flexbox

Flexbox is a one-dimensional layout system — it lays items in a row or column. Set display: flex on a container and all direct children become flex items.

Container Properties
.container {
  display:         flex;
  flex-direction:  row;    /* or column */
  justify-content: center; /* main axis */
  align-items:     center; /* cross axis */
  flex-wrap:       wrap;
  gap:             1rem;
}
Item Properties
.item {
  flex:       1;      /* grow and fill */
  flex-grow:  2;      /* grow 2× faster */
  flex-shrink:0;      /* do not shrink */
  flex-basis: 200px; /* starting size */
  align-self: flex-end;
  order:      -1;    /* move to front */
}
Memory trick: justify-content aligns along the main axis (usually horizontal). align-items aligns along the cross axis (usually vertical). When flex-direction: column, they swap.

§7CSS Grid

Grid is a two-dimensional layout system — it works with rows and columns at the same time. It is the best tool for building full page layouts.

/* Define the grid */
.layout {
  display:               grid;
  grid-template-columns: 1fr 3fr;      /* sidebar + main (25/75) */
  grid-template-rows:    60px auto 50px;/* header, content, footer */
  gap:                   1rem;
}

/* Named template areas — very readable */
.page {
  display: grid;
  grid-template-areas:
    "header  header"
    "sidebar main  "
    "footer  footer";
  grid-template-columns: 200px 1fr;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

/* Item spanning multiple columns */
.wide-item { grid-column: span 2; }
.full-row  { grid-column: 1 / -1; }

/* Responsive grid — no media query needed */
.card-grid {
  display:               grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap:                   1.5rem;
}
/* auto-fit creates as many columns as fit the container */
/* minmax: each column is at least 220px, grows to fill */

§8Media Queries

A media query applies CSS rules only when a specific condition is true — usually a viewport width range.

/* Basic syntax */
@media (min-width: 768px) {
  /* Rules applied only on 768px+ screens */
  .sidebar { display: block; }
}

/* Common breakpoints (mobile-first, min-width) */
/* xs: base styles (below 480px) — no query needed */
@media (min-width: 480px)  { /* sm: large phones */ }
@media (min-width: 768px)  { /* md: tablets */ }
@media (min-width: 1024px) { /* lg: desktops */ }
@media (min-width: 1280px) { /* xl: large screens */ }

/* Combining conditions */
@media (min-width: 768px) and (max-width: 1023px) { /* tablets only */ }

/* Other media features */
@media (orientation: landscape)         { /* sideways */ }
@media (prefers-color-scheme: dark)     { /* dark mode */ }
@media (prefers-reduced-motion: reduce) { /* disable animations */ }
@media (hover: none)                    { /* touch device, no hover */ }

§9Responsive Web Design Principles

Responsive Web Design (RWD) was coined by Ethan Marcotte in 2010. The goal is one website that works well on every device — phone, tablet, and desktop.

Pillar 1 — Fluid Grids

Use percentages (or fr units) instead of fixed pixels for column widths. Columns resize proportionally.

/* Fixed — breaks on small screens */
.sidebar { width: 300px; }

/* Fluid — scales with the container */
.sidebar { width: 30%; }

/* Best — CSS Grid with fr units */
.layout {
  grid-template-columns: 1fr 3fr;
}
Pillar 2 — Flexible Images

Images should never overflow their container. One line of CSS is all you need as a baseline:

img {
  max-width: 100%;   /* never wider than container */
  height:    auto;   /* preserve aspect ratio */
}

/* Responsive video wrapper (16:9) */
.video-wrap {
  position:       relative;
  padding-bottom: 56.25%;
  height:         0;
}
.video-wrap iframe {
  position: absolute;
  width: 100%; height: 100%;
}
Pillar 3 — Media Queries

Apply different CSS rules at different viewport widths. Already covered in §8 above — they are the third pillar that ties fluid grids and flexible images together.

Responsive Units Cheat Sheet
%

Percentage of parent element's size

rem

Root font-size (usually 16px). Scales with user preferences.

em

Current element's font-size. Compounds through nesting.

vw

1% of viewport width. 100vw = full screen width.

vh

1% of viewport height. 100vh = full screen height.

fr

Fractional unit for Grid. 1fr = one share of free space.

ch

Width of "0" character. Good for line-length limits.

clamp

clamp(min, ideal, max) — fluid value with limits.

§10Mobile-First Design

Mobile-first means you write your base CSS for small screens first, then use min-width media queries to progressively enhance the layout for larger screens.

Mobile-First (✓ Recommended)
/* Base — mobile (no query) */
.cards {
  grid-template-columns: 1fr;
}
.sidebar { display: none; }

/* Tablet and up */
@media (min-width: 768px) {
  .cards {
    grid-template-columns: 1fr 1fr;
  }
  .sidebar { display: block; }
}

/* Desktop and up */
@media (min-width: 1024px) {
  .cards {
    grid-template-columns: repeat(3, 1fr);
  }
}
Desktop-First (✗ Avoid)
/* Base — desktop (no query) */
.cards {
  grid-template-columns: repeat(3, 1fr);
}

/* Tablet — now restrict */
@media (max-width: 1023px) {
  .cards {
    grid-template-columns: 1fr 1fr;
  }
}

/* Mobile — restrict again */
@media (max-width: 767px) {
  .cards {
    grid-template-columns: 1fr;
  }
  .sidebar { display: none; }
}
Mobile devices download all desktop CSS then override it — wasteful. Google also uses mobile-first indexing for SEO.
Mobile-First Checklist
Include <meta name="viewport" content="width=device-width, initial-scale=1.0"> on every page
Set img { max-width: 100%; height: auto; } globally
Use * { box-sizing: border-box; } globally
Write base styles for mobile first — single column, no sidebar
Use min-width in media queries only (not max-width)
Touch targets (buttons, links) must be at least 44×44px
Font size ≥ 16px on inputs — prevents iOS auto-zoom on focus
No horizontal scroll at any viewport width
Use flexible units (%, rem, fr, vw) instead of fixed px widths
Test on a real device, not just a resized browser window
