CSS3 Basics

Introduction to CSS3

What is CSS3?

CSS3 is the latest evolution of Cascading Style Sheets. It is divided into independent modules — each module can progress independently (e.g. Selectors, Animations, Flexbox, Grid). This allows browsers to implement features incrementally.

CascadeStyles applied in priority order: browser default → user → author
SpecificityInline (1000) > ID (100) > Class (10) > Element (1)
InheritanceSome properties (font, color) inherit from parent by default
!importantOverrides all specificity — use sparingly

Linking CSS to HTML

/* 1. External stylesheet (preferred) */
<link rel="stylesheet" href="style.css">

/* 2. Internal style block */
<style>
  body { background: #f5f5f5; }
</style>

/* 3. Inline style (lowest reusability) */
<p style="color: red;">Text</p>

/* CSS Variables (Custom Properties) */
:root {
  --primary: #6366f1;
  --gap: 1rem;
}
h1 { color: var(--primary); }
CSS3 new features: Custom Properties, Calc(), Transitions, Animations, Flexbox, Grid, Media Queries, Pseudo-elements.

Selectors, Specificity & Pseudo-classes

Selector Types

/* Universal */
* { box-sizing: border-box; }

/* Element */
p { margin: 0; }

/* Class */
.card { padding: 1rem; }

/* ID */
#header { height: 60px; }

/* Attribute */
input[type="email"] { border: 2px solid blue; }
a[href^="https"] { color: green; }  /* starts with */
a[href$=".pdf"]  { color: red; }    /* ends with */
a[href*="vtu"]   { font-weight:700;}/* contains */

/* Combinators */
div p    { }  /* descendant */
div > p  { }  /* direct child */
h1 + p   { }  /* adjacent sibling */
h1 ~ p   { }  /* general sibling */

/* Grouping */
h1, h2, h3 { font-family: 'Space Mono'; }

Pseudo-classes & Pseudo-elements

/* Pseudo-classes — element STATE */
a:hover    { color: blue; }
a:active   { opacity: .7; }
a:visited  { color: purple; }
input:focus { outline: 2px solid #6366f1; }
input:disabled { opacity: .4; }
input:checked  { accent-color: green; }

/* Structural */
li:first-child  { font-weight: 700; }
li:last-child   { border: none; }
li:nth-child(2n) { background: #f5f5f5; } /* even */
li:nth-child(3n+1) { color: red; }
p:not(.special)  { color: gray; }

/* Pseudo-elements — VIRTUAL nodes */
p::before {
  content: "→ ";
  color: #6366f1;
}
p::after  {
  content: " ←";
}
::selection { background: #6366f1; color: white; }
::placeholder { color: #aaa; }

Box Model, Layout & Positioning

The CSS Box Model

Every element is a rectangular box made of four concentric layers:

Margin
20px
Border
4px
Padding
16px
Content
120×60px
/* box-sizing changes HOW width is calculated */
/* content-box (default): width = content only */
.box { box-sizing: content-box; width: 200px; padding: 20px; }
/* Total width = 200 + 40 (padding) + border = 240+ */

/* border-box (recommended): width includes padding+border */
* { box-sizing: border-box; }
.box { width: 200px; padding: 20px; }
/* Total width = exactly 200px */

Positioning

/* static — default, in normal flow */
.elem { position: static; }

/* relative — offset from own normal position */
.elem { position: relative; top: 10px; left: 20px; }

/* absolute — removed from flow, positioned */
/* relative to nearest positioned ancestor */
.parent { position: relative; }
.child  { position: absolute; top: 0; right: 0; }

/* fixed — relative to viewport (stays on scroll) */
.navbar { position: fixed; top: 0; width: 100%; }

/* sticky — relative until scroll threshold */
.sidebar { position: sticky; top: 20px; }

/* z-index — stacking order (only on positioned) */
.modal    { position: fixed; z-index: 1000; }
.backdrop { position: fixed; z-index: 999; }
display: blockFull width, new line (div, p, h1)
display: inlineFlows with text, no width/height (span, a)
display: inline-blockInline flow but accepts width/height
display: noneRemoved from layout entirely
visibility: hiddenInvisible but still takes space
overflowvisible / hidden / scroll / auto

CSS3 Layout

Flexbox Layout

Container Properties

.container {
  display: flex;  /* or inline-flex */

  /* Main axis direction */
  flex-direction: row;            /* default → */
  flex-direction: row-reverse;    /* ← */
  flex-direction: column;         /* ↓ */
  flex-direction: column-reverse; /* ↑ */

  /* Wrapping */
  flex-wrap: nowrap;   /* default, no wrap */
  flex-wrap: wrap;     /* wrap to next line */

  /* Align on MAIN axis (justify) */
  justify-content: flex-start;    /* default */
  justify-content: flex-end;
  justify-content: center;
  justify-content: space-between; /* even gaps */
  justify-content: space-around;
  justify-content: space-evenly;

  /* Align on CROSS axis (align) */
  align-items: stretch;   /* default */
  align-items: flex-start;
  align-items: flex-end;
  align-items: center;
  align-items: baseline;

  gap: 1rem;   /* shorthand for row-gap + column-gap */
}

Item Properties

.item {
  /* flex shorthand: grow shrink basis */
  flex: 1;          /* = flex: 1 1 0 */
  flex: 0 0 200px;  /* fixed 200px, no grow/shrink */
  flex: 1 1 auto;   /* grow and shrink from auto size */

  flex-grow: 1;    /* how much to grow to fill space */
  flex-shrink: 1;  /* how much to shrink if overflow */
  flex-basis: auto;/* starting size before growing */

  /* Override container's align-items for this item */
  align-self: auto | flex-start | flex-end | center | stretch;

  /* Re-order visual position (not DOM order) */
  order: 0;   /* default. lower = earlier */
  order: -1;  /* move to front */
  order: 2;   /* move towards end */
}
Memory trick: justify = main axis (usually horizontal). align = cross axis (usually vertical). When direction is column, they swap!

CSS Grid Layout

Grid Container

.container {
  display: grid;

  /* Define columns */
  grid-template-columns: 200px 1fr 1fr;
  grid-template-columns: repeat(3, 1fr);
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));

  /* Define rows */
  grid-template-rows: 80px auto 60px;
  grid-auto-rows: minmax(100px, auto);

  /* Named areas */
  grid-template-areas:
    "header header header"
    "sidebar main   main"
    "footer  footer footer";

  /* Spacing */
  gap: 1rem;           /* row and column gap */
  row-gap: 1rem;
  column-gap: 2rem;

  /* Alignment */
  justify-items: start | end | center | stretch;
  align-items:   start | end | center | stretch;
}

Grid Item Placement

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

/* Manual placement by line numbers */
.item {
  grid-column: 1 / 3;    /* span cols 1–2 */
  grid-column: 1 / -1;   /* span ALL columns */
  grid-column: span 2;   /* span 2 cols wide */
  grid-row:    1 / 3;    /* span rows 1–2 */
}

/* Per-item alignment */
.item {
  justify-self: start | end | center | stretch;
  align-self:   start | end | center | stretch;
}

/* fr = fractional unit */
/* 1fr = 1 fraction of available space */
/* repeat(auto-fill) = as many cols as fit */
/* minmax(min, max) = size within bounds */
Grid vs Flexbox: Use Grid for two-dimensional layouts (rows AND columns). Use Flexbox for one-dimensional (a row OR a column). They complement each other — Grid for page structure, Flexbox for component internals.

Media Queries

Media Query Syntax

/* Basic syntax */
@media media-type and (condition) {
  /* CSS rules */
}

/* Width breakpoints */
@media (max-width: 768px) { ... }   /* tablets down */
@media (min-width: 769px) { ... }   /* desktop up */
@media (min-width: 480px) and
       (max-width: 768px) { ... }   /* tablets only */

/* Media types */
@media screen  { ... }  /* screens */
@media print   { ... }  /* printing */
@media speech  { ... }  /* screen readers */
@media all     { ... }  /* default — all types */

/* Feature queries */
@media (orientation: landscape) { ... }
@media (orientation: portrait)  { ... }
@media (hover: hover)  { ... }  /* has mouse */
@media (hover: none)   { ... }  /* touch device */
@media (prefers-color-scheme: dark) { ... }
@media (prefers-reduced-motion: reduce) { ... }
@media (min-resolution: 2dppx) { ... }  /* retina */

Standard Breakpoints

xs: < 480pxSmall phones (portrait)
sm: 480–767pxLarge phones (landscape)
md: 768–1023pxTablets
lg: 1024–1279pxSmall desktops / laptops
xl: ≥ 1280pxLarge desktops
/* Mobile-first breakpoints (min-width) */
/* Base styles — mobile */
.nav { display: none; }

@media (min-width: 768px) {  /* tablet+ */
  .nav { display: flex; }
}
@media (min-width: 1024px) { /* desktop+ */
  .nav { font-size: 1.1rem; }
}

/* CSS in HTML <link> */
<link rel="stylesheet" href="mobile.css">
<link rel="stylesheet"
      href="desktop.css"
      media="(min-width: 1024px)">

Responsive Web Design

Responsive Design Principles

The Three Pillars of RWD

Ethan Marcotte coined Responsive Web Design in 2010. It rests on three technical foundations:

1
Fluid Grids
Use percentage-based widths instead of fixed pixels. Columns resize proportionally with the viewport.
2
Flexible Images
Images scale within their container using max-width: 100%. Never overflow their parent.
3
Media Queries
Apply different CSS rules at different viewport sizes. Adapt layout, typography, and spacing.

Viewport Meta Tag

Without this tag, mobile browsers render the page at a virtual desktop width (980px) and scale it down. Always include it:

<meta name="viewport"
      content="width=device-width, initial-scale=1.0">

/* width=device-width: use actual device width */
/* initial-scale=1.0: no initial zoom */

/* Avoid user-scalable=no — accessibility issue */
/* Avoid maximum-scale=1 — traps keyboard users */
em / remRelative to font-size — scales with user preferences
vw / vh% of viewport width / height
vmin / vmaxSmaller / larger of vw and vh
chWidth of the "0" character — great for text
clamp(min,val,max)Fluid value between min and max

Fluid Grids & Flexible Images

Fluid Grid Formulas

Converting fixed-pixel grids to fluid (percentage) grids:

/* Formula: target ÷ context × 100 = % */
/* E.g. 960px layout, 300px sidebar */
/* 300 ÷ 960 × 100 = 31.25% */

.container  { max-width: 1200px; width: 100%; margin: 0 auto; }
.sidebar    { width: 31.25%; float: left; }
.main       { width: 65.625%; float: right; }

/* Modern approach — CSS Grid */
.layout {
  display: grid;
  grid-template-columns: 1fr 3fr;  /* 25% / 75% */
  gap: 2rem;
}

/* Fully fluid responsive grid */
.card-grid {
  display: grid;
  grid-template-columns:
    repeat(auto-fit, minmax(250px, 1fr));
  gap: 1.5rem;
}
/* auto-fit: create as many cols as fit */
/* minmax: min 250px, grow to fill 1fr */

/* Fluid typography */
h1 { font-size: clamp(1.5rem, 4vw, 3rem); }
p  { font-size: clamp(1rem, 2vw, 1.25rem); }

Flexible Images & Media

/* Basic flexible image */
img { max-width: 100%; height: auto; }

/* Responsive image techniques */
/* 1. srcset — different sizes for different screens */
<img
  src="small.jpg"
  srcset="small.jpg 480w,
           medium.jpg 768w,
           large.jpg 1200w"
  sizes="(max-width: 480px) 100vw,
          (max-width: 768px) 50vw,
          33vw"
  alt="Description">

/* 2. picture element — art direction */
<picture>
  <source media="(max-width: 600px)"
          srcset="crop-mobile.jpg">
  <img    src="full-desktop.jpg"
          alt="Hero image">
</picture>

/* Responsive video / iframe */
.video-wrap {
  position: relative;
  padding-bottom: 56.25%; /* 16:9 ratio */
  height: 0;
}
.video-wrap iframe {
  position: absolute; inset: 0;
  width: 100%; height: 100%;
}

Mobile-First Design Approach

Mobile-First vs Desktop-First

Mobile-First ✓
Desktop-First ✗
Write base CSS for small screens
Write base CSS for large screens
Use min-width to enhance upward
Use max-width to restrict downward
Performance: mobile loads less CSS
Mobile downloads unused desktop rules
Forces content priority decisions
Easy to forget mobile adaptations
Better for SEO — Google uses mobile-first indexing
Can result in poor mobile UX

Mobile-First Code Pattern

/* Step 1: Base — smallest screen (mobile) */
.nav {
  display: flex;
  flex-direction: column;
  padding: 1rem;
}
.nav-links { display: none; }   /* hidden on mobile */
.hamburger { display: block; }  /* show burger */

.card-grid {
  display: grid;
  grid-template-columns: 1fr; /* single col */
  gap: 1rem;
}

/* Step 2: Enhance for tablets */
@media (min-width: 768px) {
  .nav { flex-direction: row; }
  .nav-links { display: flex; }  /* show links */
  .hamburger { display: none; }
  .card-grid { grid-template-columns: 1fr 1fr; }
}

/* Step 3: Enhance for desktops */
@media (min-width: 1024px) {
  .card-grid {
    grid-template-columns: repeat(3, 1fr);
  }
}

/* Step 4: Large screens */
@media (min-width: 1280px) {
  .container { max-width: 1200px; margin: 0 auto; }
}

Mobile-First Design Checklist

Touch targets ≥ 44×44px — fat-finger friendly
Single column layout as base — no sidebars
Viewport meta tag on every page
Images: max-width:100%; height:auto
min-width breakpoints only (not max-width)
No horizontal scroll on small screens
Font size ≥ 16px — prevents iOS zoom on focus
Hamburger navigation for mobile
Test on real devices, not just resizing browser
Avoid hover-only interactions (touch has no hover)
