Audio & Video Elements

📺 <video> Element

HTML5 introduced native video playback without plugins. The browser chooses the first supported <source> format.

<video width="640" height="360"
       controls autoplay muted
       loop poster="thumb.jpg"
       preload="metadata">
  <source src="clip.mp4"  type="video/mp4">
  <source src="clip.webm" type="video/webm">
  <track kind="subtitles"
         src="subs.vtt"
         srclang="en" default>
  Fallback text for old browsers.
</video>
controlsShow native play/pause/seek UI
autoplayStart playback automatically (needs muted)
mutedStart with audio off (required for autoplay)
loopRestart when ended
posterImage shown before play
preloadnone / metadata / auto
playsinlineInline on iOS (no forced fullscreen)

🔊 <audio> Element

Identical attribute set to video; no width/height/poster. Always provide MP3 + OGG for broad compatibility.

<audio controls preload="auto">
  <source src="track.mp3"
          type="audio/mpeg">
  <source src="track.ogg"
          type="audio/ogg">
  <source src="track.wav"
          type="audio/wav">
  Browser not supported.
</audio>

Canvas API

🎨 Canvas Basics

The <canvas> element is a bitmap drawing surface. All drawing is done through a 2D (or WebGL) context obtained via JavaScript.

/* HTML */
<canvas id="c"
        width="600"
        height="400">
  Fallback text
</canvas>

/* JS — get context */
const canvas = document.getElementById('c');
const ctx = canvas.getContext('2d');
fillStyleFill colour/gradient/pattern
strokeStyleStroke colour
lineWidthStroke thickness in px
fontCSS font string for text
globalAlphaTransparency 0.0 – 1.0
shadowBlurShadow blur radius

🖌️ Drawing Methods

/* Rectangles */
ctx.fillRect(x, y, w, h);
ctx.strokeRect(x, y, w, h);
ctx.clearRect(x, y, w, h);

/* Paths */
ctx.beginPath();
ctx.moveTo(50, 50);
ctx.lineTo(200, 150);
ctx.arc(cx, cy, r, 0, Math.PI*2);
ctx.closePath();
ctx.fill();
ctx.stroke();

/* Text */
ctx.font = '24px Bebas Neue';
ctx.fillText('Hello VTU', 50, 80);
ctx.strokeText('Outline', 50, 120);

/* Images */
const img = new Image();
img.onload = () => ctx.drawImage(img,0,0);
img.src = 'photo.jpg';

/* Gradients */
const g = ctx.createLinearGradient(
  0, 0, 200, 0);
g.addColorStop(0, '#6366f1');
g.addColorStop(1, '#10b981');
ctx.fillStyle = g;

/* Transform */
ctx.save();
ctx.translate(100, 100);
ctx.rotate(Math.PI / 4);
ctx.restore();

Geolocation API

📍 How It Works

The Geolocation API exposes the device's physical location via GPS, Wi-Fi triangulation, or IP lookup. Always requires explicit user permission.

// One-time position
navigator.geolocation
  .getCurrentPosition(
    (pos) => {
      const lat = pos.coords.latitude;
      const lng = pos.coords.longitude;
      const acc = pos.coords.accuracy; // metres
      console.log(lat, lng, acc);
    },
    (err) => console.error(err.message),
    {
      enableHighAccuracy: true,
      timeout: 5000,       // ms
      maximumAge: 0        // no cache
    }
  );

// Watch continuously
const id = navigator.geolocation
  .watchPosition(successFn, errorFn);

// Stop watching
navigator.geolocation.clearWatch(id);

📦 Position Object Properties

coords.latitudeDecimal degrees (−90 to +90)
coords.longitudeDecimal degrees (−180 to +180)
coords.accuracyAccuracy in metres
coords.altitudeMetres above sea level (nullable)
coords.speedm/s (nullable)
coords.heading0–360° clockwise from north
timestampDOMTimeStamp of position fix
Error Codes (PositionError.code)
1 — PERMISSION_DENIEDUser blocked access
2 — POSITION_UNAVAILABLENo fix available
3 — TIMEOUTRequest timed out

Web Storage API

💾 localStorage vs sessionStorage

localStorage
sessionStorage
Persists after tab/browser close
Cleared when tab closes
Shared across all tabs (same origin)
Isolated per tab
No expiry
Session lifetime only
~5–10 MB per origin
~5 MB per tab
window.localStorage
window.sessionStorage
Both vs Cookies: Storage is client-side only — never sent with HTTP requests (unlike cookies). Cookies max 4 KB; Storage up to 10 MB.

🔧 Web Storage API Methods

/* Write */
localStorage.setItem('key', 'value');
// Objects must be stringified
localStorage.setItem('user',
  JSON.stringify({ name: 'Arjun' }));

/* Read */
const val = localStorage.getItem('key');
const user = JSON.parse(
  localStorage.getItem('user'));

/* Delete */
localStorage.removeItem('key');
localStorage.clear(); // wipe all

/* Iterate */
for (let i = 0; i < localStorage.length; i++) {
  const k = localStorage.key(i);
  console.log(k, localStorage.getItem(k));
}

/* Storage event (cross-tab) */
window.addEventListener('storage', (e) => {
  console.log(e.key, e.oldValue, e.newValue);
});

Web Workers

⚙️ What Are Web Workers?

JavaScript is single-threaded. Web Workers run scripts in background threads, keeping the UI responsive during heavy computation.

// main.js — create a worker
const worker = new Worker('worker.js');

// Send data to worker
worker.postMessage({ n: 40 });

// Receive result
worker.onmessage = (e) => {
  console.log('Result:', e.data);
};

// Handle errors
worker.onerror = (e) => {
  console.error(e.message);
};

// Terminate when done
worker.terminate();

📁 worker.js (separate file)

// worker.js — runs in background thread
self.onmessage = (e) => {
  const n = e.data.n;
  const result = fibonacci(n);

  // Send result back to main thread
  self.postMessage({ result, n });
};

function fibonacci(n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

// Workers CANNOT access:
// - window, document, DOM
// - localStorage (use IndexedDB)
// Workers CAN use:
// - fetch, XMLHttpRequest
// - setTimeout, setInterval
// - WebSockets, IndexedDB
// - importScripts()
Dedicated Workernew Worker('file.js') — one tab only
Shared Workernew SharedWorker() — multiple tabs
Service WorkerProxy/cache — works offline too

Offline Web Apps & AppCache

📶 AppCache (Application Cache)

AppCache lets pages work offline by declaring files to cache in a .appcache manifest. Deprecated in modern browsers — replaced by Service Workers.

<!-- HTML: link manifest -->
<html manifest="app.appcache">

--- app.appcache ---
CACHE MANIFEST
# v1.0 — comment to force update

CACHE:
  index.html
  style.css
  app.js
  logo.png

NETWORK:
  *           # allow all other URLs

FALLBACK:
  /api/ offline.html
⚠️ AppCache is deprecated in HTML5.1+ and removed from modern browsers. Use Service Workers for production apps.

🔧 Service Worker (Modern Replacement)

// main.js — register SW
if ('serviceWorker' in navigator) {
  navigator.serviceWorker
    .register('/sw.js')
    .then(reg => console.log('SW registered'))
    .catch(err => console.error(err));
}

// sw.js — intercept fetch
const CACHE = 'vtu-v1';
const FILES = ['/', '/style.css', '/app.js'];

self.addEventListener('install', e => {
  e.waitUntil(
    caches.open(CACHE)
      .then(c => c.addAll(FILES))
  );
});

self.addEventListener('fetch', e => {
  e.respondWith(
    caches.match(e.request)
      .then(cached => cached ||
        fetch(e.request))
  );
});
installCache files on first register
activateClean up old caches
fetchIntercept network requests
pushReceive push notifications

Bitmap drawing. getContext('2d'). Paths, shapes, text, images, gradients. Game loop with requestAnimationFrame.

Vector XML. Elements: rect, circle, path, text. Scales losslessly. DOM accessible — CSS/JS events work directly.

💾

localStorage (permanent) vs sessionStorage (tab-scoped). Methods: setItem, getItem, removeItem, clear. Strings only.

Background thread. new Worker('file.js'). postMessage() to communicate. Cannot access DOM. Terminate with .terminate().

Deprecated. Declare CACHE, NETWORK, FALLBACK sections. Use Service Workers instead for offline-first PWAs.
