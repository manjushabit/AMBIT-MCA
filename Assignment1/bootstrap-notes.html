§1What is Bootstrap & Why Use It?

Bootstrap is a free, open-source CSS framework originally created by Mark Otto and Jacob Thornton at Twitter in 2011. It gives you a ready-made set of CSS classes and JavaScript components so you can build responsive, good-looking pages quickly — without writing all the CSS from scratch.

⚡
Speed

Ready-made components mean you build a nav bar in minutes instead of hours. Prototype fast, iterate faster.

📱
Responsive by default

Bootstrap's 12-column grid is mobile-first. Every component works on any screen size out of the box.

🌐
Cross-browser

Bootstrap handles browser inconsistencies for you. It includes Normalize.css which resets browser defaults.

Adding Bootstrap to a project
<!-- Method 1: CDN (easiest — no install needed) -->
<!-- Add to <head> -->
<link rel="stylesheet"
      href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css">

<!-- Add before </body> (needed for interactive components) -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>

<!-- Method 2: npm (for production projects) -->
npm install bootstrap
// In your JS entry point:
import 'bootstrap/dist/css/bootstrap.min.css'
Bootstrap 5 vs Bootstrap 4: Bootstrap 5 dropped jQuery as a dependency, improved the grid system, added a new utility API, and introduced RTL support. Always use Bootstrap 5 for new projects.

§2The Bootstrap Grid System

Bootstrap divides the page into 12 columns. You tell each element how many columns it should span. The key classes are container, row, and col-*.

<!-- Three-column layout -->
<div class="container">         <!-- centres content, adds side padding -->
  <div class="row">              <!-- a horizontal group of columns -->
    <div class="col-md-4">Column 1</div>  <!-- 4/12 = 33% on md+ -->
    <div class="col-md-4">Column 2</div>
    <div class="col-md-4">Column 3</div>
  </div>
</div>

<!-- Responsive: full width on mobile, 3-col on desktop -->
<div class="col-12 col-md-4">...</div>

<!-- Sidebar + main layout -->
<div class="col-md-3">Sidebar</div>   <!-- 3/12 = 25% -->
<div class="col-md-9">Main</div>      <!-- 9/12 = 75% -->

§4Bootstrap Utilities

Utilities are single-purpose classes that set one CSS property. Instead of writing custom CSS for small adjustments, you add a utility class directly in HTML.

Spacing — Margin & Padding
<!-- Pattern: {property}{sides}-{size} -->
<!-- property: m=margin, p=padding -->
<!-- sides: t=top, b=bottom, s=start/left, e=end/right, x=left+right, y=top+bottom -->
<!-- size: 0, 1(4px), 2(8px), 3(16px), 4(24px), 5(48px), auto -->

class="mt-3"    → margin-top: 1rem (16px)
class="mb-4"    → margin-bottom: 1.5rem
class="px-3"    → padding-left + right: 1rem
class="py-2"    → padding-top + bottom: 0.5rem
class="ms-auto" → margin-left: auto (pushes right)
class="mx-auto" → centre horizontally
class="p-0"     → padding: 0 (remove all padding)

§6Customising Bootstrap with Sass

Sass (Syntactically Awesome Stylesheets) is a CSS preprocessor — it adds features like variables, nesting, mixins, and functions to CSS, then compiles to regular CSS. Bootstrap 5 is entirely built with Sass.

Why Sass over CSS variables? Bootstrap uses Sass variables at compile time — changing $primary regenerates every button, badge, alert, and link colour in one go. CSS custom properties only change the visual output, not the compiled class names.
Sass Basics (what Bootstrap uses)
// 1. Variables ($ prefix)
$primary: #1d4ed8;
$border-radius: 0.5rem;
$font-size-base: 1rem;

// 2. Nesting
.card {
  padding: 1rem;
  &:hover {              // & = parent selector
    box-shadow: 0 4px 12px rgba(0,0,0,.15);
  }
  .card-title {          // compiles to .card .card-title
    font-weight: 700;
  }
}

// 3. Mixins (reusable blocks)
@mixin flex-centre {
  display: flex;
  align-items: centre;
  justify-content: centre;
}
.hero { @include flex-centre; }

// 4. Functions
$half: math.div(100%, 2);   // = 50%
// darken(), lighten(), rgba() are common
How to Customise Bootstrap — Step by Step
// custom.scss — YOUR file

// Step 1: Override Bootstrap Sass variables BEFORE importing
$primary:       #1d4ed8;    // your brand blue
$secondary:     #475569;
$success:       #16a34a;
$danger:        #dc2626;
$border-radius: 0.5rem;    // rounder buttons/cards
$font-size-base:0.9375rem; // 15px body text
$font-family-sans-serif: 'Plus Jakarta Sans', system-ui;
$headings-font-weight:    700;
$spacer:        1rem;       // base for all spacing utilities
$enable-shadows:true;       // add subtle shadows to components

// Step 2: Import Bootstrap source (not the compiled CSS)
@import "node_modules/bootstrap/scss/bootstrap";

// Step 3: Add your own extra styles AFTER
.navbar {
  box-shadow: 0 1px 8px rgba(0,0,0,.12);
}
.btn {
  font-weight: 600;
  letter-spacing: 0.02em;
}

Without Sass (CDN users): If you cannot compile Sass, override Bootstrap using CSS custom properties after the Bootstrap CDN link. For example :root { --bs-primary: #1d4ed8; }. This is less complete than Sass but works without a build step.
