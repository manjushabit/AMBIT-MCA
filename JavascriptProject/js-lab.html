JavaScript Basics
Variables
Variables are named containers that store values. JavaScript has three ways to declare them: var, let, and const.
Keyword Scope Reassignable? Hoisted? Use when
var Function Yes Yes (as undefined) Old code only — avoid in new code
let Block { } Yes No (TDZ error) Values that will change (loops, counters)
const Block { } No No (TDZ error) Values that won't change — use by default
Best practice: Default to const. Switch to let only when you need to reassign. Never use var in modern code.

§5
Control Structures
if / else / switch
Control structures decide which code runs based on conditions. They are the decision-making backbone of every program.
if-else syntax reference 
// if — runs if condition is truthy
if (condition) {
  // code runs when condition is true
}

// if-else
if (score >= 90) {
  console.log("Grade A");
} else if (score >= 75) {
  console.log("Grade B");
} else if (score >= 60) {
  console.log("Grade C");
} else {
  console.log("Fail");
}

// switch — checks one value against many cases
switch (day) {
  case "Monday":  console.log("Week starts"); break;
  case "Friday":  console.log("Weekend soon"); break;
  default:       console.log("Regular day");
}

Loops
Loops repeat a block of code. JavaScript has for, while, do...while, for...of, and for...in loops.
LoopUse whenKey feature
for You know the number of iterations Counter with init, condition, increment
while Loop until a condition becomes false Checks condition BEFORE each iteration
do...while Run at least once, then check conditionChecks condition AFTER each iteration
for...of Iterate over array elements (values) Clean syntax, no index needed
for...in Iterate over object keys (properties) Returns the key name, not value

Functions
Functions
A function is a reusable block of code that does a specific job. You define it once and call it whenever you need it.
four ways to write a function 
// 1. Function Declaration — hoisted, named
function greet(name) {
  return `Hello, ${name}!`;
}

// 2. Function Expression — not hoisted
const add = function(a, b) {
  return a + b;
};

// 3. Arrow Function — concise, no own 'this'
const multiply = (a, b) => a * b;

// 4. Arrow with body (when multiple lines needed)
const describe = (name, score) => {
  const grade = score >= 75 ? "Pass" : "Fail";
  return `${name}: ${grade}`;
};

§8

Scope
Scope controls where a variable can be accessed. JavaScript has global scope, function scope, and block scope.
Scope typeAccessible where
Global Anywhere in the script
Function Only inside the function
Block Only inside { } (let/const only)
var leaks out of blocks! That's why we prefer let and const. They respect the { } boundary.
